üë§ User Commands

Commands available to all group members.

/start User

Start the bot and get a welcome message with basic information.

Example: 
/start
/verify User

Initiate the verification process. Displays QR code and button for biometric verification.

Example: 
/verify
What happens:
  • Bot sends QR code and verification button
  • User scans QR or taps button to open Mercle app
  • User completes face verification
  • Bot unmutes user upon successful verification
/status User

Check your current verification status.

Example: 
/status
/help User

Display help message with available commands and bot information.

Example: 
/help
/rules User

View the group rules (if set by admins).

Example: 
/rules

üëÆ Admin Commands

Commands available only to group administrators.

Moderation Commands

/vkick @user Admin Only

Kick a user from the group. User can rejoin with invite link.

Usage:
  • Reply to user's message with /vkick
  • Or use /vkick @username
  • Or use /vkick <user_id>
Example: 
/vkick @spammer
/vban @user [reason] Admin Only

Permanently ban a user from the group. User cannot rejoin.

Parameters:
  • @user - User to ban (required)
  • reason - Reason for ban (optional)
Examples: 
/vban @spammer
/vban @user Repeated spam violations
/vverify @user Admin Only

Manually verify a user, bypassing the biometric verification process.

Example: 
/vverify @user

Settings Commands

/settings Admin Only

View current group settings or update specific settings.

Available Settings:
  • /settings - View all settings
  • /settings timeout <seconds> - Set verification timeout
  • /settings location <group/dm/both> - Set verification location
  • /settings autoverify <on/off> - Toggle auto-verification
Examples: 
/settings
/settings timeout 120
/settings location dm
/settings autoverify on
/setrules <text> Admin Only

Set or update the group rules.

Example: 
/setrules 1. Be respectful
2. No spam
3. Stay on topic
/stats Admin Only

View verification statistics for the group.

Example: 
/stats

Greeting Commands

/setwelcome <message> Admin Only

Set a custom welcome message for new members. Supports variables and buttons.

Variables:
  • {name} - User's first name
  • {mention} - Mention the user
  • {group} - Group name
Button Format:
  • [Button Text](URL)
Example: 
/setwelcome Welcome {mention} to {group}! [Rules](https://example.com/rules)
/setgoodbye <message> Admin Only

Set a goodbye message for leaving members.

Example: 
/setgoodbye Goodbye {name}, hope to see you again!
/goodbye <on/off> Admin Only

Enable or disable goodbye messages.

Examples: 
/goodbye on
/goodbye off
/welcome Admin Only

Test the welcome message (preview).

Example: 
/welcome

Filter Commands

/filter <keyword> <response> Admin Only

Create an auto-response filter for a specific keyword.

Examples: 
/filter hello Hello! Welcome to our group!
/filter rules Check our [Rules](https://example.com/rules)
/filters Admin Only

List all active filters in the group.

Example: 
/filters
/stop <keyword> Admin Only

Remove a filter.

Example: 
/stop hello

Lock Commands

/lock <type> Admin Only

Lock a specific message type. Locked messages are automatically deleted.

Available Types:
  • links - Block all URLs
  • media - Block photos, videos, files
  • stickers - Block stickers and GIFs
  • forwards - Block forwarded messages
Examples: 
/lock links
/lock media
/lock stickers
/unlock <type> Admin Only

Unlock a message type.

Example: 
/unlock links
/locks Admin Only

Show all active locks in the group.

Example: 
/locks

Notes Commands

/save <notename> <content> Admin Only

Save a note that can be retrieved later.

Examples: 
/save rules Please read our rules at example.com
/save faq Check our [FAQ](https://example.com/faq)
/get <notename> User

Retrieve a saved note. Can also use #notename.

Examples: 
/get rules
#rules
/notes User

List all saved notes in the group.

Example: 
/notes
/clear <notename> Admin Only

Delete a saved note.

Example: 
/clear rules

Warning Commands

/warn @user [reason] Admin Only

Warn a user. After 3 warnings, user is automatically kicked.

Examples: 
/warn @user
/warn @user Please follow group rules
/warnings @user Admin Only

Show all warnings for a user.

Example: 
/warnings @user
/resetwarns @user Admin Only

Clear all warnings for a user.

Example: 
/resetwarns @user

Whitelist Commands

/whitelist add @user Admin Only

Add a user to the whitelist. Whitelisted users bypass verification.

Example: 
/whitelist add @trusteduser
/whitelist remove @user Admin Only

Remove a user from the whitelist.

Example: 
/whitelist remove @user
/whitelist list Admin Only

Show all whitelisted users in the group.

Example: 
/whitelist list

Admin Log Commands

/adminlog Admin Only

View recent admin actions in the group.

Usage:
  • /adminlog - View last 20 actions
  • /adminlog @user - View actions for specific user
  • /adminlog <user_id> - View actions by user ID
Examples: 
/adminlog
/adminlog @user
