Biometric Verification

Secure face verification powered by Mercle SDK. Prevent bots, fake accounts, and spam with advanced biometric authentication.

Key Features

  • Face Recognition: 99.9% accuracy using AWS Face Liveness technology
  • Auto-Mute: New members are automatically muted until verified
  • Flexible Location: Choose where verification happens (group, DM, or both)
  • Configurable Timeout: Set custom verification timeout (default: 2 minutes)
  • Global Verification: Once verified, users are verified across all groups

How It Works

1

New Member Joins

Bot automatically mutes the user and sends verification prompt

2

User Verifies

User scans QR code or taps button to open Mercle app

3

Face Scan

Mercle app performs liveness check and face verification

4

Instant Access

User is unmuted and can participate in the group

Configuration

/settings location <group/dm/both>  # Set verification location
/settings timeout <seconds>         # Set verification timeout
/settings autoverify <on/off>      # Toggle auto-verification

Admin Commands

  • /vverify @user - Manually verify a user
  • /settings - View and configure verification settings

Admin Tools

Comprehensive moderation commands for managing your Telegram group effectively.

Moderation Commands

/vkick @user Admin Only

Kick a user from the group. They can rejoin if they have the invite link.

Example: 
/vkick @spammer
/vban @user [reason] Admin Only

Permanently ban a user from the group. They cannot rejoin.

Example: 
/vban @spammer Repeated spam violations
/warn @user [reason] Admin Only

Warn a user. After 3 warnings, user is automatically kicked.

Example: 
/warn @user Please follow group rules

Whitelist Management

Bypass verification for trusted members:

/whitelist add @user       # Add user to whitelist
/whitelist remove @user    # Remove from whitelist
/whitelist list            # Show all whitelisted users

Settings Management

Configure group-specific settings:

/settings                  # View current settings
/settings timeout 120      # Set 2-minute timeout
/settings location dm      # Verify in DM
/settings autoverify on    # Enable auto-verification

Welcome/Goodbye System

Greet new members and say goodbye to leaving members with customizable messages and interactive buttons.

Welcome Messages

Create custom welcome messages with support for variables and inline buttons.

Variables

  • {name} - User's first name
  • {mention} - Mention the user
  • {group} - Group name

Button Format

Add inline buttons using markdown-style syntax:

[Button Text](https://example.com/url)

Example

/setwelcome Welcome {mention} to {group}! Please read our [Rules](https://example.com/rules) and [Verify](https://t.me/yourbot?start=verify) your identity.

Goodbye Messages

Send farewell messages when users leave:

/setgoodbye Goodbye {name}, hope to see you again!
/goodbye on                # Enable goodbye messages
/goodbye off               # Disable goodbye messages

Testing

Test your welcome message before going live:

/welcome                   # Show welcome message preview

Message Filters

Auto-respond to specific keywords with custom messages. Perfect for FAQs and common questions.

Creating Filters

/filter <keyword> <response>

Examples

/filter hello Hello! Welcome to our group!
/filter rules Check our rules [Rules](https://example.com/rules)
/filter support Need help? [Contact Support](https://example.com/support)

Managing Filters

/filters                   # List all active filters
/stop <keyword>            # Remove a filter

Use Cases

  • FAQ Automation: Auto-respond to common questions
  • Welcome Messages: Greet users who say "hello"
  • Resource Links: Provide quick access to documentation
  • Support Routing: Direct users to support channels

Features

  • ‚úì Unlimited filters per group
  • ‚úì Support for inline buttons
  • ‚úì Case-insensitive matching
  • ‚úì Substring matching (keyword anywhere in message)

Message Locks

Restrict specific message types to keep your group clean and focused. Locked messages are automatically deleted.

Available Lock Types

  • links - Block all URLs and links
  • media - Block photos, videos, and files
  • stickers - Block stickers and GIFs
  • forwards - Block forwarded messages

Commands

/lock <type>               # Lock a message type
/unlock <type>             # Unlock a message type
/locks                     # Show current locks

Examples

/lock links                # Block all links
/lock media                # Block all media
/unlock stickers           # Allow stickers again
/locks                     # View active locks

Use Cases

  • Prevent Spam: Lock links to prevent spam
  • Text-Only Discussions: Lock media for focused conversations
  • Professional Groups: Lock stickers for serious discussions
  • Anti-Promotion: Lock forwards to prevent channel promotion

Admin Exceptions

Notes & Tags System

Save and retrieve group notes with hashtag support. Perfect for storing rules, FAQs, and important information.

Saving Notes

/save <notename> <content>

Examples

/save rules Please read our rules at example.com/rules
/save faq Check our FAQ [FAQ Page](https://example.com/faq)
/save welcome Welcome to our community!

Retrieving Notes

Two ways to retrieve notes:

/get <notename>            # Using command
#notename                  # Using hashtag

Examples

/get rules                 # Get rules note
#rules                     # Same as above
#faq                       # Get FAQ note

Managing Notes

/notes                     # List all saved notes
/clear <notename>          # Delete a note

Advanced Features

  • Media Support: Save notes with images or files
  • Button Support: Add inline buttons to notes
  • Hashtag Retrieval: Quick access with #notename
  • Per-Group Storage: Each group has its own notes

Use Cases

  • Store group rules and guidelines
  • Save frequently asked questions
  • Quick access to important links
  • Store welcome messages and templates

Admin Logs

Track all admin actions with detailed logging. Perfect for accountability and audit trails.

What Gets Logged

  • User kicks and bans
  • Warnings issued
  • Manual verifications
  • Whitelist additions/removals
  • Settings changes

Viewing Logs

/adminlog                  # View recent admin actions
/adminlog @user            # View actions for specific user
/adminlog <user_id>        # View actions by user ID

Log Format

Each log entry includes:

  • Action type (kick, ban, warn, etc.)
  • Admin who performed the action
  • Target user
  • Reason (if provided)
  • Timestamp

Anti-Flood Protection

Prevent spam and flooding with intelligent rate limiting and auto-moderation.

How It Works

The bot tracks message frequency and automatically takes action when users exceed configured thresholds.

Default Thresholds

  • Message Rate: 10 messages per 10 seconds
  • Action: Auto-mute for 5 minutes
  • Repeat Offenders: Automatic kick after 3 violations

Features

  • ‚úì Automatic detection of rapid messaging
  • ‚úì Configurable thresholds per group
  • ‚úì Escalating penalties for repeat offenders
  • ‚úì Admin exemption (admins not affected)
  • ‚úì Automatic unmute after timeout

Configuration

Anti-flood is enabled by default. Contact support for custom threshold configuration.

Use Cases

  • Prevent message spam attacks
  • Stop bot flooding
  • Limit rapid-fire messaging
  • Protect group from raids
