2Ô∏è‚É£ Set Bot Permissions

The bot needs admin permissions to function properly. Here's what permissions are required and why:

Required Permissions

Delete Messages Required

Needed to clean up verification messages and delete locked message types.

Ban Users Required

Needed for /vban command and to kick users who fail verification.

Restrict Members Required

Needed to mute unverified users and manage member permissions.

Pin Messages Optional

Optional, but useful for pinning important messages like rules.

How to Grant Permissions

  1. Open your group in Telegram
  2. Tap the group name at the top
  3. Tap "Administrators"
  4. Find the bot in the list
  5. Tap "Edit" next to the bot
  6. Enable the required permissions listed above
  7. Tap "Save" or the checkmark
‚ö†Ô∏è Important: Without proper permissions, the bot cannot mute/unmute users or perform moderation actions.

3Ô∏è‚É£ Configure Bot Settings

Customize the bot's behavior for your group:

View Current Settings

/settings

This will show all current settings for your group.

Common Configuration Options

Set Verification Timeout

How long users have to complete verification (in seconds):

/settings timeout 120      # 2 minutes (default)
/settings timeout 180      # 3 minutes
/settings timeout 300      # 5 minutes

Set Verification Location

Where verification messages are sent:

/settings location group   # In group chat (default)
/settings location dm      # In private message
/settings location both    # In both locations
üí° Tip: Use dm location to keep your group chat clean. The bot will send a small notice in the group and the full verification in DM.

Toggle Auto-Verification

Whether to automatically verify new members:

/settings autoverify on    # Enable (default)
/settings autoverify off   # Disable

Set Welcome Message

Create a custom welcome message for new members:

/setwelcome Welcome {mention} to {group}! Please read our [Rules](https://example.com/rules) and verify your identity.

Set Group Rules

Set rules that users can view with /rules:

/setrules 1. Be respectful to all members
2. No spam or advertising
3. Stay on topic
4. No hate speech or harassment

4Ô∏è‚É£ Test First Verification

Before going live, test the verification flow:

Testing Steps

  1. Create a test account (or use a friend's account)
  2. Add the test account to your group
  3. Observe the bot's behavior:
    • Bot should mute the new member
    • Bot should send verification prompt (QR code + button)
    • Location depends on your settings (group/DM)
  4. Complete verification:
    • Scan QR code with Mercle app
    • Or tap the "Open Mercle App" button
    • Complete face verification
  5. Verify success:
    • User should be unmuted
    • Bot should send success message
    • User can now send messages

Manual Verification (For Testing)

If you want to bypass verification for testing:

/vverify @testuser

Check Verification Status

Users can check their status with:

/status

üîß Troubleshooting

Bot Not Responding

Problem: Bot doesn't respond to commands
Solutions:
  • Check if bot is still in the group
  • Verify bot has admin permissions
  • Try removing and re-adding the bot
  • Make sure you're using commands in the group (not DM)

Bot Can't Mute Users

Problem: New members aren't being muted
Solutions:
  • Verify bot has "Restrict Members" permission
  • Check that auto-verification is enabled: /settings autoverify on
  • Make sure the bot is an admin

Verification Messages Not Appearing

Problem: No verification prompt when users join
Solutions:
  • Check verification location setting: /settings
  • If set to "dm", check user's DM with the bot
  • Verify bot has permission to send messages
  • Try manually triggering: /verify

Users Can't Complete Verification

Problem: Users stuck at verification step
Solutions:
  • Ensure users have the Mercle app installed
  • Check if QR code is displaying correctly
  • Try increasing timeout: /settings timeout 300
  • Use manual verification as fallback: /vverify @user

Bot Deleting Wrong Messages

Problem: Bot deleting messages it shouldn't
Solutions:
  • Check active locks: /locks
  • Unlock unwanted locks: /unlock <type>
  • Review message filters: /filters
  • Admins are exempt from locks
